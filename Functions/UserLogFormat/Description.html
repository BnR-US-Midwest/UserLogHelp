Description

This function performs the same operation as UserLogQuick with the addition of formatting runtime data. The string message can include the % identifier to place runtime data. The following data types are included in the args structure.

  • %b BOOL
  • %f LREAL (double)
  • %i DINT (signed long)
  • %s STRING

There are up to USERLOG_FORMAT_INDEX + 1 entries of each argument type.

Depending on the UserLog's current severity level, set by UserLogSetSeverityLevel, this function's message can be suppressed. If suppressed, nothing is written to the logbook and the function returns zero.

The size (or length) of the string passed to message is unrestricted because it is passed by reference. This is denoted by STRING[0]. Messages in the UserLog library are truncated by USERLOG_MESSAGE_LENGTH.

The Event ID created for the record entry in the User logbook uses the user's severity and code, then the facility USERLOG_FACILITY. To use a custom facility, see UserLogMessage.

If an error occurs during execution, an automatic message will be written to the User logbook with a description of the error. The function will return zero.
