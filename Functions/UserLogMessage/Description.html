Description

The UserLogMessage function provides all features of the ArEventLogWrite function block and more, synchronously. The record's event ID is create from severity, facility, and code. An origin ID is optionally available to create a record hierarchy. message is written to the additional log data with arEVENTLOG_ADDFORMAT_TEXT. object ID provides a optional description from the creator of the record.

The timestamp of the record is automatically set with the time at the call of this function. If object is left NULL, UserLogMessage will automatically read the software object (program) from which this function was called.

Depending on the UserLog's current severity level, set by UserLogSetSeverityLevel, this function's message can be suppressed. If suppressed, nothing is written to the logbook and the function returns zero.

Just as in UserLogFormat, an arguments structure can be passed with identifies in message to automatically format runtime data.

If an error occurs during execution, an automatic message will be written to the User logbook with a description of the error. The function will return zero.
