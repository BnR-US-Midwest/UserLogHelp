Technical Information

UserLog's source code is written in C. The implementation utilizes ArEventLog's function blocks to find, create, and write to user logbooks. Instead of function blocks, which require multiple calls, UserLog includes functions to use as a single synchronous call.

It is not possible to write to any Automation Runtime system logbook with either UserLog or ArEventLog. Automation Runtime creates one logbook named "User", the $arlogusr identifier, that permits user writes. This is done automatically with UserLogBasic and UserLogAdvanced.

Users can create logbooks in an Init routine using UserLogCreate and subsequently writing to those logbooks with UserLogCustom.

UserLog is built for SG4 systems.

UserLog is open source an available on GitHub. New releases can be viewed here.
