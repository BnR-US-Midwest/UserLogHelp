Technical Information

UserLog's source code is written in C. The implementation utilizes ArEventLog's function blocks to find, create, and write to user logbooks. Instead of function blocks, which require multiple calls, UserLog provides functions for single synchronous call use.

It is not possible to write to system logbooks with either UserLog or ArEventLog libraries. Automation Runtime creates one logbook labelled "User", internally named $arlogusr, that permits user writes. Messages are automatically written to this logbook with UserLogBasic and UserLogAdvanced.

Users can create logbooks in an Init routine using UserLogCreate and subsequently writing to those logbooks with UserLogCustom.

UserLog is built for SG4 systems.

UserLog is open source an available on GitHub. New releases can be viewed here.
