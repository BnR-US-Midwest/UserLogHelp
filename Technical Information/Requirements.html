Requirements

UserLog reports function errors to the User logbook. The list of possible errors is as follows.

Error Constant Event ID (Text ID) Description
USERLOG_ERROR_IDENT -515234032 UserLog could not identify the logbook by the name provided. Please see ArEventLog error.
USERLOG_ERROR_WRITE -515234022 UserLog could not write to the logbook. Please see ArEventLog error.
USERLOG_ERROR_CREATE -515234012 UserLog could not create the logbook. Please see ArEventLog error.

For each error, the logbook record entry will include the specific ArEventLog error number encountered during runtime.

The UserLog error reporting requires the UserLog.tmx User Event Texts file to be deployed to the active configuration's Textsystem Configuration.

This is achieved by first adding a Project.language file in Logical View, then adding and configuring a TC.textconfig file in the Physical View.

These instructions can also be followed for any User Event Texts file.

Project Language

In Logical View, navigate to the Toolbox to find a Project Languages object.


Text System Configuration

In Configuration View, select the TextSystem package and choose Textsystem Configuration from the Toolbox.

Set the language fields and then select UserLog.tmx from the dropbox in the next available field of the tmx files list.
